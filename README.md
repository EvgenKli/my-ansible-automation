# Configuration Management & Kubernetes Bare-Metal Cluster Deploy

Репозиторий содержит сценарии Ansible для автоматической настройки операционных систем и развертывания инфраструктурного ПО на "голом железе" (Bare-Metal).

## Состав сценариев (Playbooks)
1.  **`deploy_postgres.yml`** — Идемпотентный сценарий развертывания СУБД PostgreSQL 16. Автоматизирует установку, конфигурирует системные лимиты, файлы сетевой безопасности (`pg_hba.conf`), инициализирует пользователя и базу данных для бэкенда.
2.  **`install_k8s_tools.yml`** — Сценарий подготовки нод кластера Kubernetes. Отключает Swap, активирует сетевые фильтры ядра Linux (`br_netfilter`), разворачивает контейнерный движок `Containerd` (с настройкой Systemd Cgroup) и фиксирует версии утилит `kubeadm/kubelet/kubectl` через `dpkg hold`.

## Сетевые особенности и автоматизация
*   **Jump Host (Bastion):** Настроена маршрутизация Ansible через сервер (Мастер-ноду) с помощью параметров `ProxyCommand` в файле `hosts`. Это позволяет конфигурировать Воркер-ноду, скрытую в приватной сети.
*   **Доверенные ключи:** Сценарий автоматически связывает ключи операционных систем Мастера и Воркера для их беспрепятственного автономного общения внутри кластера.

## Инструкция по запуску
1.  Актуализируйте IP-адреса серверов в файле `hosts`.
2.  Запустите быструю проверку связи (ad-hoc ping):
    ```bash
    ansible all -i hosts -m ping
    ```
3.  Запустите плейбук настройки СУБД или компонентов Kubernetes:
    ```bash
    ansible-playbook -i hosts deploy_postgres.yml
    ansible-playbook -i hosts install_k8s_tools.yml
```
